# Subscription Manager 使用说明

## 日常使用入口

生产环境推荐通过 Hermes 自然语言交互。Web UI 主要用于首次管理员初始化、API Token 管理、可选币种/显示偏好、应急检查和批量人工核对。

可以直接对 Hermes 说：

- “列出未来 30 天要续费或到期的订阅。”
- “查一下 ChatGPT Plus 的价格、下次续费日、自动续费状态和提醒规则。”
- “下个月预计续费多少？分别按原币种列出，并给出人民币估算。”
- “把某订阅改为不再自动续费，服务到期日是 2026-09-30。”
- “给某订阅增加提前 7 天和 1 天提醒。”
- “记录今天支付 USD 20，并推进到下一账期。”
- “显示最近的付款和审计记录。”

## 读取与写入确认

查询通常直接执行。以下操作必须由 Hermes 复述最终参数并等待明确确认：

- 新建订阅；
- 修改价格、币种、周期、自动续费或关键日期；
- 计划取消或撤销取消；
- 设置提醒规则；
- 记录付款及决定是否推进账期；
- 归档或恢复订阅。

如果名称匹配多条记录、日期/币种/金额不明确或订阅版本已变化，Hermes 应停止并让用户选择/确认，不会自行猜测。

## 关键概念

- **自动续费**：是否预计供应商会在下一周期继续扣费。关闭后不显示下一续费日，也不生成后续账单事件。
- **计划取消**：记录服务将在某日失效并停止后续账单计划；不会自动去供应商网站完成取消。
- **服务到期日**：当前已购买权益真正失效的日期。
- **取消截止日**：必须在该日前操作以避免下一周期扣费。
- **付款推进**：当前 planned billing event 的正常付款可推进一次账期；历史补录不应推进。
- **归档**：停止日常跟踪但保留计划、付款和审计历史，可恢复，不是硬删除。归档后不再计入未来续费、总览预测、Upcoming Events 或提醒；历史实际支出仍保留。

## 提醒如何工作

Scheduler 根据订阅事件和规则生成可靠 Outbox。Hermes 的唯一周期任务领取到期项，通过 Hermes 已配置的渠道发送；发送成功才确认，失败则退避重试。系统停机后的短期遗漏会补发，重复扫描不会重复创建同一提醒。

若没有收到提醒，依次检查：

1. 订阅是否有未来账单/到期事件；
2. reminder rule 是否启用、offset 是否正确；
3. Scheduler 是否运行且 `NOTIFICATION_MODE=external`；
4. Hermes Reminder Consumer 最近是否成功运行；
5. Token 是否有效并包含 `reminders:consume`；
6. Hermes 最终通知渠道是否可用；
7. audit/log 中是否存在 fail、dead 或过期租约。

不要通过手动重复 ack 或启动第二个消费者解决提醒问题。

## 金额与汇率

原始金额始终按币种分别统计。Dashboard/统计可使用 ECB 最新工作日参考汇率交叉估算人民币，并显示汇率日期。ECB 不可达或缺少某币种汇率时，不会生成不完整的人民币合计；原币种统计仍可用。

## 账号和应急入口

管理员可在 Web UI“设置”中修改密码；修改后所有浏览器 Session 都会撤销。忘记密码时必须登录主机执行本地交互式重置，不提供邮件找回：

```bash
docker compose --env-file .env -f deploy/compose.production.yml -p subscription-manager \
  exec backend python -m app.cli reset-admin-password --username admin
```

API Token 遗失时不能找回明文。应创建新 Token、更新 Hermes secret、验证后撤销旧 Token。

## Web UI 页面

- **总览**：下月按币种续费总额、人民币汇率估算和近期事件。
- **订阅**：搜索、筛选、创建、编辑、关键日期、付款、提醒、取消计划、归档/恢复。
- **未来事件**：账单、试用结束、服务到期、取消截止和合同结束。
- **统计**：预计/实际支出以及供应商、分类维度。
- **设置**：主题、时区、默认提醒、可选币种、管理员密码和 API Token。

日常 Hermes 交互出现疑问时，可用 Web UI 只读核对；两者访问同一 API 和数据库。
